LeapFrame Configuration File

            Leapframe requires a base configuration file, leapframe.xml,  to be included in the config  directory.  This configuration file has two required sections; Properties, and Handlers.

Properties
           
            Properties are attributes added to the application manager object.  The application manager is instantiated into the application scope.  Any static, application specific, property should be included here.  The required properties are:

  • viewpath – this should be the path to the directory that contains the view components.
  • configpath – the directory from the root directory that contains the xml configuration files.
  • Httproot – Uri to the root of the application. ( i.e. http://www.leapframe.com)

 

            The properties tag must be included in the leapframe.xml file and must have at least one child property tag.  The property tag has two required attributes: name, and value.

            <properties>
                        <property name=”[property name]” value=”[property value]” />
            </properties>

Handlers

Includes
            Includes are a way in which you may subdivide your application's action handlers into  functionality groupings.  For instance, if your site has a section for user management and a shopping cart, you could divide the actions into userHandlers.xml, and shoppingCartHandlers.xml.   This allows you to focus on only the task you are working on currently without having to constantly step over the other sections handlers.  This does introduce an issue, however.  In larger systems it may become difficult to track down in all of your handler files where the handler for a particular action may be.  For this reason I have, by convention, appended a section prefix to every action.  For example, within the userHandlers.xml file I define all handler actions as “user.[actionName]”.  The include tag has one required attribute: xml. The value passed to the xml attribute should be the path to the file you are including without the filetype extension.  The path you include will be relative to the configpath attribute specified in index.cfm.

            <include xml=”[includefilepath]” />
